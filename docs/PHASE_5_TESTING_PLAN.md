## 1️⃣ 認證流程測試 (Authentication Flow)

### 1.1 Credentials 登入 (稅號 + 密碼)
```
✅ 測試：有效稅號 + 正確密碼 → 成功登入
✅ 測試：無效稅號 → 返回 null / 401
✅ 測試：有效稅號 + 錯誤密碼 → 返回 null / 401
✅ 測試：帳號狀態為 INACTIVE → 拒絕登入 / 403
✅ 測試：帳號狀態為 SUSPENDED → 拒絕登入 / 403
✅ 測試：空白稅號 / 密碼 → 返回 400
```

**驗證點**：
- [ ] 密碼雜湊驗證成功
- [ ] 登入成功返回 JWT token
- [ ] 登入失敗返回適當錯誤碼
- [ ] 帳號狀態檢查正常
- [ ] 無 SQL injection 漏洞

**測試帳號**：
- 稅號：`12345678` / 密碼：`Admin1234!` / 角色：SUPER_ADMIN
- 稅號：`99998888` / 密碼：`Admin1234!` / 角色：ADMIN

### 1.2 OAuth 流程 (Google + Apple)
```
✅ 測試：首次 Google 登入（新用戶）→ 重定向至註冊頁面
✅ 測試：Google 帳戶已連結 → 直接登入
✅ 測試：Google OAuth 帳戶連結至現有用戶 → 成功連結
✅ 測試：Apple OAuth 流程 → 同上
✅ 測試：OAuth token 過期 → 自動刷新
```

**驗證點**：
- [ ] OAuth provider 回調正常
- [ ] 用戶信息正確同步
- [ ] 多次登入不重複建立帳號
- [ ] email 驗證流程

### 1.3 Bearer Token 驗證 (移動應用)
```
✅ 測試：有效 JWT Token → 返回用戶信息
✅ 測試：過期 Token + 在寬限期內 → 自動刷新
✅ 測試：過期 Token + 超過寬限期 → 拒絕 / 401
✅ 測試：無效簽名 → 拒絕 / 401
✅ 測試：缺少 user ID → 拒絕 / 401
✅ 測試：用戶已刪除 → 拒絕 / 404
```

**驗證點**：
- [ ] Token 簽名驗證
- [ ] Token 過期時間檢查
- [ ] 寬限期邏輯（7 天）
- [ ] 最大年限檢查（60 天）

### 1.4 Session 驗證 (Web 應用)
```
✅ 測試：有效 Session → 返回用戶信息
✅ 測試：無效 Session → 返回 null / 401
✅ 測試：Session 過期 → 重新登入
✅ 測試：用戶不存在 → 返回 null
✅ 測試：帳號已停用 → 拒絕訪問
```

**驗證點**：
- [ ] Session 儲存正常
- [ ] Session 過期檢查
- [ ] Cookie 設定正確
- [ ] CSRF token 保護
